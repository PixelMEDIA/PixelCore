PixelCore

Getting Started Guide

Intro

PixelCore is a boilerplate solution for ASP.NET MVC 4. It includes several common utility libraries and base classes to help speed up web development.

Solution boilerplate

The included solution follows a CompanyName.ProjectName namespacing convention. A new application solution would be created from scratch, with the following projects:

PixelMEDIA.PixelCore
Utilities, attributes, and interfaces.
PixelMEDIA.PixelWeb
General base classes and helpers for various presentation-layer objects such as Razor views and controllers.
PixelMEDIA.*
Any other shared projects that may be useful for this application.
(CompanyName.ProjectName).Core
Common items for the whole application. We separate this out from BusinessLogic to avoid circular project references in some application-specific utility libraries.
(CompanyName.ProjectName).BusinessLogic
The main project for application logic.
(CompanyName.Projectname).Components
Application-specific base classes and helpers for various presentation-layer objects such as Razor views and controllers.
(CompanyName.ProjectName).Web
The main MVC web application project.

Usually, we create the application-specific projects fresh and bring in the basic code for the (CompanyName.Projectname).Components library from a previous project (now this boilerplate).

Setting base classes in the MVC project

In the CompanyName.ProjectName.Web project Views folder Web.config, set the pageBaseType like the following:

<pages pageBaseType="(CompanyName.ProjectName).Components.BaseClasses.(ProjectName)WebViewPage">

After this change, you may need to restart Visual Studio to have intellisense see the @Pixel property in Razor pages. This is set up to support plain and strongly-typed views.

For controllers, simply inherit from the (ProjectName)Controller class you created in the (CompanyName.ProjectName).Components project.

AppSettings

Use the ApplicationSettings class in (CompanyName.ProjectName).Core to stop using magic strings across your application to retrieve appSettings values from your web.config. You can then access these centrally defined, strongly typed properties from ApplicationSettings.Instance or from AppSettings properties attached to controllers and the @Pixel helper class.

@Pixel.AppSettings.SomeWebConfigProperty
